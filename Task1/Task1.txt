Задание 1. Анализ безопасности системы

1.1. Выявите конфиденциальные данные, которые не учтены во внутренних системах

    Компания обрабатывает следующие категории конфиденциальных данных, требующих защиты 
    в соответствии с ФЗ-152 "О персональных данных" и отраслевыми требованиями:

    - PII (Personally Identifiable Information):
    ФИО, дата рождения, телефон, email
    Адрес прописки, место работы/учёбы
    - Медицинские данные:
    Диагнозы, результаты анализов, медицинские карты (PDF/JPG)
    Хронические заболевания, анамнез
    - Финансовые данные:
    Платёжные реквизиты (через ККМ и 1С)
    Договоры на оказание услуг
    - Учётные данные сотрудников:
    Логины/пароли (Active Directory)
    Данные для зарплатного учёта (1С:Бухгалтерия)

  Диграммы потоков данных, ссылка: https://drive.google.com/file/d/11AA3t_SK5c2Gu7Jsnv6LiKe5xIrIGrSy/view?usp=sharing

    Вкладки: 
        - Процесс записи пациента на приём   
            Риски:
                - Утечка данных из Excel (нет RBAC)
                - Нет шифрования при передаче
        - Процесс обработки медицинских данных 
            Риски:
                - Мед. данные не отделены от финансовых
                - Нет контроля версий файлов
        - Процесс учёта ТМЦ
            Риски:
                - Доступ к ТМЦ есть у всех (нет ролей "Склад"/"Бухгалтерия").

2.2. Проведите аудит мер по обеспечению безопасности данных

    Проблемы текущего учёта:
    - Данные хранятся в неструктурированном виде (Excel, сканы) без разграничения доступа
    - Нет аудита операций с данными
    - Интеграции между системами (1С, Excel, сервер) не защищены

3.3. Подумайте, что можно улучшить

    Категоризация данных:
        - Выделить домены: PII, Medical, Financial, HR
        - Для каждого домена определить RBAC-роли (например, "Врач" → доступ только к своим пациентам)
    Инструменты:
        - Заменить Excel на СУБД (Postgres для структурированных данных, MinIO для файлов)
        - Внедрить Vault для управления секретами (API-ключи лаборатории)
    Безопасность:
        - Шифрование данных в rest и transit (TLS, PGP)
        - Data Loss Prevention (DLP) для мониторинга утечек
    Интеграции:
        - API-шлюз (Kong) для контроля доступа к лаборатории
        - Аудит логгирование (Elasticsearch + SIEM)

    Диаграмма: https://drive.google.com/file/d/11AA3t_SK5c2Gu7Jsnv6LiKe5xIrIGrSy/view?usp=sharing

    (вкладка: Интеграция с лабораторией анализов)

    Требования:
        - Маскирование PII в API (например, заменять ФИО на ID)
        - Audit log для всех запросов

    Текущая система хранения данных не соответствует требованиям безопасности. 
    Необходимо внедрить централизованное хранилище с контролем доступа, аудитом и шифрованием. 
    Первый шаг — миграция с Excel на СУБД и настройка RBAC
