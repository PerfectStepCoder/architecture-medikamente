Задание 3. Оценка Data Encryption at Rest and In Transit

    3.1. Определить, какие данные нужно защитить

        Классификация данных:
            a) По типу и чувствительности
                - PII (Персональные данные) (ФИО, телефон, паспортные данные)
                - Медицинские данные (Диагнозы, анализы, медкарты)
                - Финансовые данные (Реквизиты платежей, договоры)
                - Учётные данные (Логины/пароли, доступы к системам)
                - Метаданные (Время записи, ID пациента)
            b) По состоянию данных
                - Хранимые данные (БД, файлы на серверах, бэкапы)
                - Передаваемые данные (API-запросы, передача между сервисами, загрузка в облако)
                - Данные в процессе обработки (Оперативная память, кэш приложений)
    
    3.2. Реализовать базовые инструменты защиты
        a) Шифрование
            |Тип данных|Метод защиты|Инструменты|
            |--|---|---|
            |Персональные данные|AES-256, TDE (Transparent Data Encryption)|PostgreSQL (pg_crypto), BitLocker, VeraCrypt|
            |Передаваемые данные|TLS 1.3, HTTPS, VPN|Nginx (SSL), OpenVPN, WireGuard
            |Данные в процессе обработки|Контейнеры с изоляцией памяти|Intel SGX, AWS Nitro Enclaves|
        b) Контроль доступа
            RBAC (Role-Based Access Control) — роли: врач, кассир, администратор
            ABAC (Attribute-Based Access Control) — доступ на основе атрибутов (например, «только к своим пациентам»)
            Инструменты (Keycloak (IAM), HashiCorp Vault (управление секретами))
        c) Анонимизация и маскирование
            Динамическое маскирование (например, заменять ФИО на ID в BI-отчётах)
            Tokenization — замена чувствительных данных на токены (например, платёжные реквизиты)
            Инструменты (Apache Atlas (Data Lineage), Protegrity (маскирование))
        d) Мониторинг и аудит
            SIEM-система для анализа логов и алертов (например, Elastic SIEM)
            DLP (Data Loss Prevention) — предотвращение утечек (например, Symantec DLP)
            Audit Logs — запись всех действий с данными (например, PostgreSQL Audit Extension)

    3.3. Автоматизированный контроль
        a) Политики безопасности
            - Автоматическая блокировка при множественных неудачных входах (Fail2Ban)
            - Сканирование уязвимостей (OpenVAS, Nessus)
            - Проверка серверных конфигураций
        b) Интеграция с CI/CD
            - Проверка кода на уязвимости (SonarQube, GitGuardian)
            - Тестирование на соответствие GDPR/ФЗ-152 (OWASP ZAP)


    3.4. Реестр средств защиты данных

        | Данные                     | В чём необходимость защиты                                            | Инструменты защиты                                                      |
        |----------------------------|-----------------------------------------------------------------------|-------------------------------------------------------------------------|
        | **PII (Персональные данные)** | Защита от утечки персональных данных пациентов (ФЗ-152, GDPR)      | HashiCorp Vault, PostgreSQL pg_crypto, Dynamic Masking в BI             |
        | **Медицинские записи**       | Конфиденциальность диагнозов, результатов анализов                  | AES-256 шифрование (GPG), DLP-системы, Access Control (RBAC)            |
        | **Финансовые транзакции**    | Защита платёжных реквизитов, предотвращение мошенничества           | PCI DSS-совместимые решения, Tokenization, HSM                          |
        | **Учётные данные сотрудников** | Предотвращение несанкционированного доступа к системам            | MFA (YubiKey, Google Authenticator), Keycloak IAM                       |
        | **API-запросы**             | Защита данных при передаче между сервисами                           | TLS 1.3, API Gateway с авторизацией (Kong)                      |
        | **Бэкапы**                  | Защита резервных копий от утери/кражи                                | BorgBackup с шифрованием, изолированное хранилище (Air-Gapped)          |
        | **Журналы аудита**          | Обеспечение целостности логов для расследований                      | SIEM (Wazuh, Splunk), Blockchain для логов (IBM Blockchain Platform)    |
        | **Данные в Kubernetes**     | Защита конфигураций и секретов в облачной среде                      | Sealed Secrets, KMS (AWS KMS)                          |
        | **Мобильные данные**        | Защита данных в мобильном приложении                                 | App Shield (DexGuard), Certificate Pinning                              |
        | **Электронная почта**       | Предотвращение утечки конфиденциальной информации через email        | S/MIME шифрование, ProtonMail, Exchange Online Protection               |
